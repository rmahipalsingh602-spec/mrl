Installation

Download the official Windows build, place mrl.exe in a stable tools directory, add that directory to your PATH, and run MRL from any project folder with the mrl command.

mrl --version
mrl test.mrl

In PowerShell or CMD, use mrl your-file.mrl. Typing only .mrl will not work because that is just the file extension.

Editor Setup

If the VS Code marketplace does not show MRL yet, use the local extension that already ships in this repository.

  1. Open mrl-vscode-extension/ in Visual Studio Code.
  2. Run npm install.
  3. Press F5 to launch the Extension Development Host.
  4. Open an .mrl or .hi file and use MRL: Run MRL File.
  5. For the full MRL logo on files, choose Preferences: File Icon Theme and select MRL File Icons.

What's New in MRL 3.1

  • Counted loop variables: use mrl gino i = 1 se 5 for clean range-style iteration.
  • Boolean logic: use sach, jhooth, aur, ya, and nahi.
  • Cleaner branching: add warna for a natural else block.
  • Tooling polish: updated VS Code snippets, syntax highlighting, and file icons for a more premium workflow.

Hello World

MRL programs are readable from the first line. The language prefix makes intent explicit and keeps the surface easy to scan.

mrl bolo "Hello World"

Basic Syntax

mrl boloOutput

Print text or expression results.

mrl bolo "Hello World"
mrl rakhoState

Create or update a variable.

mrl rakho score = 10
mrl agarControl

Start a conditional block and close it with khatam.

mrl agar score > 5
    mrl bolo "Ready"
khatam
warnaFallback

Add a fallback branch to an agar block.

mrl agar score >= 50
    mrl bolo "Pass"
warna
    mrl bolo "Retry"
khatam
mrl jab takLoop

Repeat while the condition stays true.

mrl jab tak score < 20
    mrl rakho score = score + 1
khatam
mrl ginoCounted Loop

Run a counted loop with an explicit loop variable and optional step.

mrl gino i = 1 se 5 kadam 2
    mrl bolo i
khatam
mrl functionReuse

Define a function with parameters.

mrl function greet(name)
    mrl bolo name
khatam
mrl callInvoke

Call a defined function with arguments.

mrl call greet("MRL")
mrl importModules

Load another module by name.

mrl import utils
mrl aiAI

Send a prompt into an AI-backed workflow.

mrl ai "Generate landing page copy"
sach / jhoothBoolean

Work with readable boolean values and logical operators.

mrl rakho ready = sach
mrl agar ready aur nahi jhooth
    mrl bolo "System ready"
khatam

Examples

The language is compact enough for quick scripts and structured enough for launch-ready runtime targets.

mrl rakho launch_ready = sach

mrl agar launch_ready aur nahi jhooth
    mrl bolo "MRL launch wave active"
warna
    mrl bolo "Launch blocked"
khatam

mrl gino wave = 1 se 5
    mrl bolo "Wave " + wave
khatam

Advanced Usage

  • Packages: install reusable modules and expand your project without leaving the MRL workflow.
  • AI workflows: combine readable commands with generated ideas, helpers, and content.
  • App runtime: build interface-driven programs through MRL app targets.
  • Web runtime: serve MRL-powered pages through the web execution path.
  • Future compiler path: bytecode, VM, and JIT milestones continue to push performance and runtime sophistication forward.
mrl install utils
mrl import utils
mrl ai "Draft a launch status update"

mrl gino pulse = 10 se 2 kadam -4
    mrl bolo "Pulse " + pulse
khatam

To start the web runtime, run mrl run web in your terminal. That command belongs in the CLI, not inside an .mrl source file.
