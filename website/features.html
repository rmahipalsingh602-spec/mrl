Flow control

Programs can say more with less ceremony.

MRL 3.1 upgrades the everyday language surface so scripts, tools, and launch demos read more like intent and less like workarounds.

  • mrl gino counted loops for variable-driven iteration
  • warna else blocks for cleaner branching
  • sach, jhooth, aur, ya, nahi for boolean logic that feels native to MRL
Runtime system

Multi-runtime from the start.

MRL still targets different execution modes without forcing users to learn different language surfaces for each one.

  • Native CLI workflows for scripts and automation
  • App runtime for interface-led experiences
  • Web runtime for MRL-powered pages and previews
Tooling polish

The language now feels better in the editor too.

Launch quality depends on more than syntax. MRL now carries a sharper workflow with stronger VS Code support and file identity.

  • Dedicated file icons for .mrl and .hi files
  • Updated snippets and keyword completion for the new syntax
  • Improved file association story for a more native Windows feel
AI and packages

Still ready for ecosystem growth.

The platform story remains bigger than syntax. Packages, prompts, GUI, and web output keep MRL moving toward a wider ecosystem.

  • Consistent command story across native, app, and web targets
